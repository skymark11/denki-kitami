# 地域カスタマイズ（ローカライズ）ワークフロー

サイトを $1（地域名）、$2（電話番号）向けにカスタマイズする。

## 前提
- CLAUDE.md を必ず先に読み込む
- 現在のサイトは「北見市」向けに最適化されている
- 差し替え対象は以下のファイル群すべて

## Step 1: 差し替えマッピングの確認

以下の値をユーザーから収集する（不足分は確認する）:

| パラメータ | 現在値（北見） | 新しい値 |
|-----------|---------------|---------|
| 地域名 | 北見市 | $1 |
| 都道府県 | 北海道 | （確認必要） |
| 電話番号 | 070-4080-0965 | $2 |
| 電話番号(ハイフンなし) | 07040800965 | （$2から生成） |
| 緯度 | 43.8044 | （確認必要） |
| 経度 | 143.9107 | （確認必要） |
| 対応エリア半径 | 30000(m) | （確認必要） |
| 周辺地域 | 網走市、美幌町 | （確認必要） |
| ドメイン | denki-kitami.com | （確認必要） |
| サイト名 | 北見の街の電気屋さん | （確認必要） |

## Step 2: ファイル別の差し替え箇所

### 2-1. 全HTMLファイル（index.html + 各サブページ/index.html）
- `<title>` 内の「北見市」→ 新地域名
- `<meta name="description">` 内の地域名
- `<meta name="keywords">` 内の地域名
- Open Graph タグ内の地域名・URL
- `<link rel="canonical">` のドメイン
- JSON-LD構造化データ:
  - `addressLocality`
  - `addressRegion`
  - `telephone`
  - `geo.latitude` / `geo.longitude`
  - `geoRadius`
  - `areaServed` の各City name
  - `url`
- 本文中の「北見市」「北見」の地域名差し替え
- 電話番号（表示用 + tel:リンク両方）
- フッターの住所・電話番号

### 2-2. 設定ファイル
- `robots.txt` — Sitemap URLのドメイン
- `sitemap.xml` — 全URLのドメイン
- `send_mail.php` — 送信先メールアドレス・ヘッダー情報
- `.htaccess` — ドメイン関連のリダイレクトルール

### 2-3. CSS/JS
- `css/style.css` — コメント内のサイト名（機能に影響なし）
- `js/script.js` — 電話番号追跡用の値があれば差し替え

## Step 3: コンテンツのローカライズ

単純な文字列置換だけでなく、以下も調整が必要:

- **寒冷地対応の記述**: 地域の気候に合わせて調整（北海道以外なら不要な場合あり）
- **対応エリアセクション**: 新地域の周辺市町村に書き換え
- **北海道特有の表現**: 「積雪対応」「寒冷地仕様」等を地域に合わせて変更/削除
- **料金相場**: 地域の物価水準に応じて調整の要否を確認

## Step 4: 新リポジトリの構成

```
denki-{地域slug}/
├── （北見版と同じ構造）
├── CLAUDE.md            ← 地域名・電話番号等を更新
└── .claude/skills/      ← Skills はそのまま継承
```

## Step 5: 検証チェックリスト
- [ ] 全ページで「北見」が残っていないこと（意図的な場合を除く）
- [ ] 電話番号が全箇所で統一されていること
- [ ] JSON-LD構造化データの座標・住所が正しいこと
- [ ] canonical URLが新ドメインを指していること
- [ ] sitemap.xmlの全URLが新ドメインであること
- [ ] OGP画像URLが正しいこと
- [ ] send_mail.phpの送信先が更新されていること
