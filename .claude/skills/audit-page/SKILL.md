# ページ品質監査ワークフロー

$ARGUMENTS で指定されたページ（ディレクトリ名またはパス）を監査する。

## 前提
- CLAUDE.md を必ず先に読み込む
- このスキルは読み取り専用（allowed-tools: Read, Grep, Glob）
- ファイルの変更は行わず、レポートのみ出力

## 監査1: SEO チェック（各項目10点満点、合計100点）

### 1-1. タイトルタグ（10点）
- [ ] 32文字以内か
- [ ] メインKWを含むか
- [ ] 形式: 「北見市の{サービス}｜{差別化}｜北見の街の電気屋さん」か
- 減点: 長すぎ(-3), KW不足(-5), 形式不一致(-2)

### 1-2. メタディスクリプション（10点）
- [ ] 120文字以内か
- [ ] メインKW + 差別化ポイントを含むか
- [ ] 電話番号を末尾に含むか
- [ ] CTAワード（「無料見積もり」等）を含むか

### 1-3. H1タグ（10点）
- [ ] ページに1つだけか
- [ ] メインKWを含むか
- [ ] titleタグと異なる表現か

### 1-4. 見出し構造（10点）
- [ ] H1→H2→H3の階層が正しいか
- [ ] H2にサブKWが含まれるか
- [ ] スキップ（H1→H3）がないか

### 1-5. 構造化データ（10点）
- [ ] JSON-LD形式か
- [ ] @type: Service または LocalBusiness が正しいか
- [ ] telephone, address, areaServed が正しいか
- [ ] FAQPage構造化データがあるか
- [ ] AggregateOffer（価格帯）が正しいか

### 1-6. 内部リンク（10点）
- [ ] トップページへのリンクがあるか
- [ ] 関連サービスページへのリンクがあるか
- [ ] パンくずリスト or ナビゲーション内にリンクがあるか

### 1-7. 画像最適化（10点）
- [ ] 全画像にalt属性があるか
- [ ] alt属性にKWが自然に含まれるか
- [ ] WebP形式が使用されているか

### 1-8. canonical URL（10点）
- [ ] `<link rel="canonical">` が存在するか
- [ ] URLが正しい（末尾スラッシュ統一）か

### 1-9. OGPタグ（10点）
- [ ] og:title, og:description, og:type, og:url, og:image が全て存在するか

### 1-10. 共起語カバレッジ（10点）
- 該当サービスの共起語分析ファイルが存在する場合:
  - 必須KW30語中の含有率を計算
  - 90%以上: 10点、80%: 8点、70%: 6点、60%以下: 4点

## 監査2: コンプライアンスチェック

### 景品表示法
- [ ] 「No.1」「最安値」「最高」等の最上級表現がないか
- [ ] 実績数値に根拠があるか（虚偽禁止）
- [ ] 「お客様満足度」等の数値に調査根拠が示されているか

### 電気工事業法
- [ ] 建設業許可の表示がないか（保有していないため禁止）
- [ ] 電気工事士資格のみを信頼性の根拠としているか

### 特定商取引法
- [ ] 事業者名が明記されているか
- [ ] 連絡先（電話番号）が明記されているか
- [ ] 価格表示が適切か（「〜」付きで目安明示）

### 個人情報保護法
- [ ] プライバシーポリシーへのリンクがあるか
- [ ] フォームに個人情報の取り扱い説明があるか

## 監査3: レスポンシブ・アクセシビリティ

- [ ] viewport metaに `maximum-scale=5.0, user-scalable=yes` があるか
- [ ] 4段階ブレークポイントのCSSが存在するか（1024, 900, 768, 480px）
- [ ] tel:リンクが正しい形式か（ハイフンなし）
- [ ] フォントサイズが16px以上か（モバイル入力ズーム防止）

## 出力形式

監査結果は以下の形式で出力:

```markdown
# {ページ名} 品質監査レポート
日付: YYYY-MM-DD

## SEOスコア: XX/100点
| 項目 | スコア | 備考 |
|------|--------|------|
| タイトルタグ | X/10 | ... |
| ... | ... | ... |

## コンプライアンス: {OK / 要改善}
- {問題点があれば列挙}

## レスポンシブ: {OK / 要改善}
- {問題点があれば列挙}

## 改善提案（優先度順）
1. {最重要の改善点}
2. ...
```

既存の監査レポートがある場合は `seo-audit/` および `compliance-audit/` を参照して、前回からの改善状況も報告する。
